核心速查（5 分钟上手）

Skill = 一份带元数据的操作手册，给 AI 的"入职指南"。

my-skill/
├── SKILL.md              # 必须：YAML frontmatter + Markdown 主体
├── scripts/              # 可选：可执行脚本
├── references/           # 可选：参考文档
└── assets/               # 可选：模板、图片、字体

Claude 启动时只把每个 Skill 的 name + description 注入上下文，匹配时再读完整内容。装 100 个 Skill，日常对话也只占 1 万词左右。

触发：自动（description 语义匹配）或手动（/skill-name）。
作用域：企业 > 个人 > 项目 > 插件。
对比：Skill 教 AI 做事（手册）；MCP 给 AI 工具（USB-C）；Sub-agent 是另一个 LLM。

frontmatter 速查表

字段必填作用
name✓Skill 名称（小写字母/数字/连字符，≤64字符）
description✓触发描述，Claude 据此判断是否调用
when_to_use补充触发条件（自然语言）
pathsglob 路径过滤，限定适用文件范围
allowed-tools工具白名单
disable-model-invocationtrue = 只允许用户手动触发
user-invocablefalse = 只允许 Claude 触发
argument-hint/ 命令参数补全提示
model覆盖使用的模型
context: fork在 subagent 隔离上下文运行
agentcontext: fork 时指定 agent 类型
hooks生命周期钩子

description + when_to_use 合计 ≤ 1536 字符。name 长度 ≤ 64 字符。

三层加载机制

L1 · 元数据层

启动时注入：所有 Skill 的 name + description

~100 词 / Skill，常驻

L2 · 主体层

匹配触发时加载：SKILL.md 完整 Markdown

建议 < 500 行

L3+ · 资源层

按需读取：scripts/、references/、assets/

无大小限制

结果

装 100 个 Skill，常驻上下文也只占 1 万词左右

按需展开，token 经济
